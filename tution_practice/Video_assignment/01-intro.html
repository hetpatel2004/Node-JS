1:-What is Node.js and where does it run?
Node.js is a runtime environment that allows JavaScript to run outside the browser. It runs on the server
2:-Is Node.js a language, framework, or runtime?
Node.js is a runtime environment, not a language or framework.
3:-What is the V8 engine?
V8 is Google's JavaScript engine that converts JavaScript into fast machine code.
4:-How does Node.js handle multiple requests?
Node.js uses a single thread and event loop to handle many requests asynchronously.
5:-Why is Node.js non-blocking?
Because it uses asynchronous operations, so it does not wait for tasks to finish.
6:-Why Node.js is popular for APIs?
Node.js is fast, lightweight, and handles many API requests at once.
7:-Why did you choose Node.js for your project?
I chose Node.js because it is scalable, fast, and uses JavaScript for both frontend and backend.
8. What is nvm and why is it used?
nvm is a Node Version Manager used to switch between multiple Node.js versions.
9. What happens when you run a JavaScript file in Node.js?
Node loads the file, executes it using the V8 engine, and runs it in the Node runtime.
10. What is process.env and how is it used?
process.env stores environment variables used for configuration.
11. Difference between window and global?
window is for browsers, global is for Node.js.
12. How does Node.js execute JavaScript outside the browser?
Using the V8 engine and Node runtime APIs.
13. Explain the Node.js runtime lifecycle.
Start → execute code → event loop runs → process exits.
14. What are modules in Node.js?
Reusable blocks of code split into separate files.
15. What is require()?
Used to import modules in Node.js.
16. Difference between exports and module.exports?
module.exports is the actual exported object; exports is its reference.
17. What is CommonJS?
A module system used by Node.js.
18. What is module caching?
Once loaded, a module is reused instead of reloaded.
19. What are circular dependencies?
When two modules depend on each other.
20. What is npm?
Node Package Manager used to install libraries.
21. What is package.json?
A file that stores project metadata and dependencies.
22. Difference between dependencies and devDependencies?
dependencies are for production, devDependencies for development.
23. What is package-lock.json?
Locks exact versions of installed packages.
24. Difference between npm install and npm ci?
npm ci is faster and used for clean production installs.
25. What is npx?
Runs npm packages without installing them globally.
26. What is the fs module?
Used to work with files and directories.
27. Difference between sync and async fs methods?
Sync blocks execution, async does not.
28. Why is async fs preferred?
It does not block the event loop.
29. How is error handling done in fs?
Using error-first callbacks or try-catch with promises.
30. How can fs block the event loop?
By using synchronous fs methods.
31. How to handle large file uploads in Node.js?
Using streams.
32. What is a Buffer?
A temporary memory area to store binary data.
33. What are Streams?
Data processing in small chunks.
34. Difference between Readable and Writable streams?
Readable reads data, Writable writes data.
35. What does pipe() do?
Connects one stream to another.
36. Difference between streams and readFile?
Streams use less memory than readFile.
37. How are streams used in real-time applications?
For efficient data transfer like video or file streaming.
